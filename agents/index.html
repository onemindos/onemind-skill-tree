Coding Agents

Tools That Write Code Using Models

// What Are Coding Agents?

Coding agents are software tools that use large language models (LLMs) to autonomously write, edit, refactor, and review code. Unlike a simple chat interface, they can read files, run shell commands, interact with git, and complete multi-step engineering tasks.

Agents connect to models via providers — a thin routing layer that handles auth, billing, and inference. The same agent can often be reconfigured to use a completely different model or provider, giving you flexibility across cost, capability, and availability constraints.

In the OneMind stack, coding agents sit in the middle tier: they are specialized workers that receive tasks from orchestrators (Hermes, OpenClaw) and execute them using the best available model for the job.

Hermes can delegate a task to Claude Code via ACP, which then calls Anthropic (or Bedrock) to complete it. OpenClaw can spin up multiple agents in parallel — e.g. one Codex worker for reasoning tasks, one Pi Agent for an edge device — then aggregate results. The orchestrators are provider-agnostic; they route based on task requirements, cost, and availability.
