Azure AI

Microsoft Cloud AI Services
☁ Overview
Azure AI (formally the Azure OpenAI Service) is Microsoft's enterprise-grade platform for deploying large language models and AI services inside the Azure cloud. Rather than hitting a shared public API, organisations create their own dedicated deployments — private, regionally isolated instances of the model running on Microsoft infrastructure. This gives teams the compliance posture, data-residency guarantees and reliability SLAs that regulated industries require.

Azure hosts OpenAI's flagship models natively, plus an expanding Model Catalog inside Azure AI Foundry that adds partner and open-source models (Anthropic Claude, Meta Llama, Mistral, Cohere, and more). Every deployment sits behind Azure's Role-Based Access Control, Microsoft Entra ID (formerly Active Directory), private virtual-network endpoints, and built-in content safety filters — so the same governance tooling you use for your databases and services applies to your AI layer automatically.

★ Why Use Azure
🔒

Enterprise Compliance

SOC 2 Type II, ISO 27001, HIPAA BAA, FedRAMP. Your model traffic stays within Azure's compliance boundary. Audit logs, data-residency guarantees, and customer-managed encryption keys (CMK) included.

👤

Active Directory / Entra ID Integration

Authenticate to models using the same Azure AD / Entra ID identities and service principals you already use across your Azure estate. No separate API-key rotation needed — use managed identities.

🛡

Built-in Content Filtering

Azure AI Content Safety runs alongside every deployment. Configurable filters for hate, violence, self-harm, and sexual content — all configurable per severity level without any extra integration work.

🌎

Global Regions

Deploy to 60+ Azure regions worldwide. Pin your inference endpoint to the same geography as your data for latency and sovereignty. Cross-region redundancy available for high-availability configs.

📈

SLA Guarantees

Azure OpenAI Service carries a 99.9% uptime SLA backed by Microsoft credits. Provisioned throughput units (PTUs) give predictable, reserved capacity rather than shared best-effort rate limits.

🔗

Private Endpoints

Expose your Azure OpenAI resource only on a private VNet — zero public internet exposure. Traffic never leaves your network perimeter. Ideal for air-gapped enterprise and government deployments.

🔨 How to Connect
01

Create an Azure OpenAI Resource

Azure Portal → "Create a resource" → Search "Azure OpenAI" → Fill subscription, resource group, region, pricing tier → Click Create. Once provisioned, navigate to the resource and open Azure AI Studio (or Azure AI Foundry) to create a deployment.

02

Deploy a Model

In Azure AI Studio → Deployments → + Create deployment → Choose the base model (e.g. gpt-4o) → Give it a deployment name (you reference this name in API calls, not the model name). Each deployment is version-pinned and has its own TPM quota.

03

Retrieve Keys & Endpoint

In the Azure Portal → your OpenAI resource → Keys and Endpoint. The endpoint URL looks like:

https://<your-resource-name>.openai.azure.com/

Two API keys are provided (rotate without downtime). You only need one at a time.

04

Set Environment Variables

AZURE_OPENAI_ENDPOINT=https://<resource>.openai.azure.com/
AZURE_OPENAI_API_KEY=<your-api-key>
AZURE_OPENAI_DEPLOYMENT=<your-deployment-name>
Note: AZURE_OPENAI_DEPLOYMENT is the custom name you gave when deploying, e.g. gpt4o-prod — not the raw model string. Different from OpenAI direct where you pass gpt-4o as the model.
05

Python — openai library with Azure config

import os
from openai import AzureOpenAI

# Initialise Azure-flavoured client
client = AzureOpenAI(
    azure_endpoint=os.environ["AZURE_OPENAI_ENDPOINT"],
    api_key=os.environ["AZURE_OPENAI_API_KEY"],
    api_version="2024-10-01-preview",   # pin the API version
)

# model= takes your DEPLOYMENT NAME, not the base model name
response = client.chat.completions.create(
    model=os.environ["AZURE_OPENAI_DEPLOYMENT"],  # e.g. "gpt4o-prod"
    messages=[
        {"role": "system",  "content": "You are a helpful assistant."},
        {"role": "user",    "content": "Explain Azure AI in one sentence."},
    ],
    max_tokens=256,
    temperature=0.7,
)

print(response.choices[0].message.content)
Tip: For Managed Identity auth (no API key), replace api_key with azure_ad_token_provider= using azure.identity.DefaultAzureCredential. This is the recommended approach for production workloads running inside Azure.
