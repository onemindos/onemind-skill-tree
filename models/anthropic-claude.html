Anthropic Claude

The Model Behind the Agent • One Model, Many Doors

Overview

Anthropic builds Claude — a family of large language models designed for safety, helpfulness, and honesty. Founded by former OpenAI researchers, Anthropic uses Constitutional AI (RLHF with a constitution of principles) to align Claude's behavior.


Claude is the primary model powering the OneMind OS stack. What makes it unique in our architecture: you can access the same model through multiple providers. This means redundancy, cost optimization, and zero vendor lock-in. One model, many doors.

Where Claude Lives — Every Door In

This is the key insight: Claude is available through 4+ different providers. Each has different auth, pricing, and trade-offs. You're never locked into one.

Anthropic Direct API
SIMPLEST
  • Endpoint: api.anthropic.com
  • Auth: API key (ANTHROPIC_API_KEY)
  • Pricing: pay-per-token, no commitment needed
  • First to get new models
  • Rate limits can be tight on free tier
AWS Bedrock
ENTERPRISE
  • Endpoint: bedrock-runtime.{region}.amazonaws.com
  • Auth: IAM (no API keys floating around)
  • Pricing: on-demand or provisioned throughput
  • VPC private endpoints (data never leaves your VPC)
  • Regional: us-east-1, us-west-2, eu-west-1, etc.
Google Vertex AI
GCP
  • Endpoint: {region}-aiplatform.googleapis.com
  • Auth: Service account / ADC
  • Pricing: pay-per-token via GCP billing
  • Good if you're already in GCP ecosystem
  • Slightly behind on model versions
Azure AI Foundry
PARTNER
  • Via Azure AI Model Catalog
  • Auth: Azure AD / API key
  • Pricing: Azure consumption-based
  • Unified billing with other Azure services
  • Limited model versions available
Provider Auth Method Pricing Latency Best For
Anthropic Direct API Key Pay-per-token Lowest Development, prototyping, indie
AWS Bedrock IAM / STS On-demand or committed Low Enterprise, compliance, multi-region
Google Vertex Service Account GCP billing Medium GCP-native shops

⚠ Pro tip: Use Bedrock as your fallback. If Anthropic direct hits rate limits (429), Bedrock uses IAM auth on a separate quota. Zero downtime.

How to Use — Code for Every Path

Anthropic Direct
# Install pip install anthropic # Set key export ANTHROPIC_API_KEY="sk-ant-..." # Python import anthropic client = anthropic.Anthropic() msg = client.messages.create( model="claude-sonnet-4-20250514", max_tokens=1024, messages=[{"role": "user", "content": "Hello"}] )
AWS Bedrock
# No pip install needed (boto3 comes with AWS CLI) # Auth via IAM (no API key!) export AWS_ACCESS_KEY_ID="AKIA..." export AWS_SECRET_ACCESS_KEY="..." export AWS_DEFAULT_REGION="us-east-1" # Python import boto3, json client = boto3.client("bedrock-runtime") resp = client.invoke_model( modelId="anthropic.claude-sonnet-4-20250514-v1:0", body=json.dumps({ "anthropic_version": "bedrock-2023-05-31", "max_tokens": 1024, "messages": [{"role": "user", "content": "Hello"}] }) )
Google Vertex AI
# Install pip install google-cloud-aiplatform anthropic[vertex] # Auth via service account export GOOGLE_APPLICATION_CREDENTIALS="/path/to/sa.json" export CLOUD_ML_REGION="us-east5" # Python (uses Anthropic SDK with Vertex backend) from anthropic import AnthropicVertex client = AnthropicVertex( region="us-east5", project_id="my-project" ) msg = client.messages.create( model="claude-sonnet-4@20250514", max_tokens=1024, messages=[{"role": "user", "content": "Hello"}] )
Claude Code — All Backends
# Default (Anthropic direct) claude # Via AWS Bedrock export CLAUDE_CODE_USE_BEDROCK=1 claude # uses your AWS credentials # Via Google Vertex export CLAUDE_CODE_USE_VERTEX=1 claude # uses your GCP service account # Via Copilot (GitHub auth) copilot --acp --stdio # routes to Claude # The same Claude Code agent — different billing door

Claude in the OneMind Stack

Claude Code
  • Native Claude agent for coding
  • Supports: Anthropic direct, Bedrock, Vertex, Copilot
  • Switch backend with env vars — same agent, different billing
  • Delegated from Hermes via delegate_task or ACP
Hermes Agent
  • Orchestrator — routes to Claude as primary model
  • Config: provider = copilot, anthropic, or bedrock
  • Fallback chain: copilot → bedrock (auto-failover)
  • Can switch models mid-session
OpenClaw
  • Multi-agent fleet orchestrator
  • Routes different agents to different Claude tiers
  • Opus for complex tasks, Sonnet for standard, Haiku for routing
  • Cost optimization via model selection
Copilot (GitHub)
  • Routes to Claude Sonnet via GitHub's infra
  • Auth: GitHub token (free with Pro/Enterprise)
  • Used by Hermes as primary provider (no API cost)
  • Rate limits can hit (429) during heavy use
