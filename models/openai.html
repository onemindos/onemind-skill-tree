OpenAI

GPT, o-series, and Codex
◕ Overview
OpenAI is the AI research company behind the most widely deployed large language models in the world. Founded in 2015, it launched the GPT (Generative Pre-trained Transformer) lineage — each generation larger and more capable than the last: GPT-1, GPT-2, GPT-3, InstructGPT, GPT-3.5, GPT-4, and the current flagship GPT-4o (omni). Their models power ChatGPT, the OpenAI API, GitHub Copilot, Azure AI, and hundreds of thousands of applications globally.

The GPT lineage focuses on fast, general-purpose language and multimodal understanding. The newer o-series (o1, o3, o4-mini) are reasoning models that use an internal chain-of-thought process before answering, dramatically improving performance on math, code, and complex logic. The Codex family was purpose-built for code generation and powers tools like Codex CLI and early GitHub Copilot. OpenAI also offers specialised models for image generation (DALL-E), speech recognition (Whisper), and embeddings.

OpenAI models are available through three major channels: the OpenAI Direct API, Azure OpenAI Service (Microsoft's enterprise deployment), and bundled via GitHub Copilot for developer tooling.

⚙ Model Families
⚡
GPT-4o
Flagship
The current default model. "Omni" — natively multimodal across text, images, and audio in a single model. Fastest of the GPT-4 family with the same intelligence as GPT-4 Turbo. Best for interactive applications.
128K context text + vision + audio fast latency gpt-4o
🔥
GPT-4 Turbo
Previous Gen
The extended-context GPT-4 variant released late 2023. Knowledge cutoff April 2024. Still capable and widely deployed. Successor is GPT-4o. Common in enterprise Azure deployments that haven't migrated yet.
128K context text + vision gpt-4-turbo
🧠
o1 / o3 / o4-mini
Reasoning
The o-series uses internal chain-of-thought "thinking time" before responding. Dramatically better at math, logic, science, and hard coding problems. o4-mini balances reasoning power with speed and cost. o3 is the strongest general-purpose reasoning model.
chain-of-thought 200K context (o3) extended thinking o1 / o3 / o4-mini
💻
Codex
Code Specialist
Code-specialized model family fine-tuned on billions of lines of public code. Powers the Codex CLI agent (codex-1) and was the engine behind early GitHub Copilot. Excels at code completion, generation, explanation, and debugging.
code-specialized powers Codex CLI agentic tasks codex-1
⚡
GPT-4o mini
Lightweight
Smallest and cheapest in the GPT-4o family. Ideal for high-volume tasks like classification, routing, summarization, and structured extraction where cost matters. Surprisingly capable for its size.
128K context lowest cost high throughput gpt-4o-mini
🎨
DALL-E 3
Image Gen
OpenAI's text-to-image model. Accepts natural language prompts and produces photorealistic or artistic images. Integrated into ChatGPT and available via the Images API. Supports 1024x1024, 1024x1792, and 1792x1024 resolutions.
text-to-image 1024–1792px dall-e-3
🎤
Whisper
Speech-to-Text
Open-source automatic speech recognition model trained on 680K hours of multilingual audio. Available via the OpenAI Transcriptions API or self-hosted. Supports 99 languages, translation to English, and timestamps.
99 languages self-hostable whisper-1
🌐 Where OpenAI Lives
🔗

OpenAI Direct API

api.openai.com — the original public API. Pay-as-you-go token pricing. Access all models including cutting-edge previews. Set OPENAI_API_KEY. No infrastructure overhead; fastest path to new model releases.

☁

Azure OpenAI Service

Managed deployments inside Microsoft Azure. Enterprise-grade: private endpoints, VNet integration, Azure RBAC, compliance certifications (HIPAA, FedRAMP). Uses deployment names instead of model IDs. Subset of OpenAI models, slightly delayed releases.

🔨

GitHub Copilot

GPT-4o and o3 bundled inside the Copilot subscription. Accessible in VS Code, JetBrains, and CLI via the Copilot API. No direct model billing — included in Copilot seat pricing. Also exposes Claude (Anthropic) as an alternative.

Provider Auth Pricing Best For
OpenAI Direct OPENAI_API_KEY Pay-per-token, cheapest Startups, developers, prototyping, latest models
Azure OpenAI Azure AD / API key + endpoint URL Pay-per-token + infra costs Enterprise, compliance, private network, regulated industries
GitHub Copilot GitHub OAuth / GITHUB_TOKEN Flat seat subscription Developers in IDE, coding agents, Copilot CLI integration
📄 How to Use
🔗 OpenAI Direct
install
pip install openai
python
from openai import OpenAI

client = OpenAI()  # reads OPENAI_API_KEY

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "user",
         "content": "Explain transformers"}
    ]
)
print(response.choices[0].message.content)
OPENAI_API_KEY = sk-proj-...
Note: Use model="o3" or "o4-mini" for reasoning tasks. Reasoning models don't support system messages — use user messages only.
☁ Azure OpenAI
python (azure)
from openai import AzureOpenAI
import os

client = AzureOpenAI(
    azure_endpoint=os.environ["AZURE_OPENAI_ENDPOINT"],
    api_key=os.environ["AZURE_OPENAI_API_KEY"],
    api_version="2024-10-21"
)

response = client.chat.completions.create(
    model="my-gpt4o-deployment",  # deployment name
    messages=[
        {"role": "user",
         "content": "Explain Azure RBAC"}
    ]
)
print(response.choices[0].message.content)
AZURE_OPENAI_ENDPOINT = https://YOUR.openai.azure.com/
AZURE_OPENAI_API_KEY = abc123...
Key diff: Azure uses deployment names, not model IDs. You create a deployment named e.g. my-gpt4o-deployment in the Azure portal and reference it by that name. Same Python SDK, different client class.
🧠 o-series Reasoning Models 
# o3 / o4-mini: streaming disabled, no system messages
response = client.chat.completions.create(
    model="o4-mini",
    messages=[
        {"role": "user",
         "content": "Prove that sqrt(2) is irrational"}
    ],
    reasoning_effort="high"  # low | medium | high
)

# Check how much thinking was done
usage = response.usage
print(f"Reasoning tokens: {usage.completion_tokens_details.reasoning_tokens}")
⌨ OpenAI in Coding Agents
💻
Codex CLI
OpenAI's own terminal coding agent. Runs codex-1 natively via OPENAI_API_KEY, or can be pointed at an Azure deployment. Supports full-auto, suggest, and interactive modes. Sandboxed shell execution.
codex-1 (native) OPENAI_API_KEY Azure support
🔨
GitHub Copilot
Uses GPT-4o as its default coding model and o3 for complex reasoning tasks and Copilot Workspace. Bundled via GitHub's infrastructure — developers don't manage API keys. Also surfaces Claude and Gemini as model choices.
GPT-4o (default) o3 reasoning GITHUB_TOKEN auth
⚡
Hermes Agent
Nous Research's orchestration agent. Configurable to route to OpenAI via OPENAI_API_KEY or through Azure OpenAI. Switch models per-skill or per-session. Part of the OneMind OS stack.
gpt-4o routing o-series support OPENAI_API_KEY Azure path
🔎
OpenClaw
Multi-agent fleet system. Can route tasks to OpenAI models including GPT-4o, o3, and o4-mini. Useful for parallelizing work across model families — fast GPT-4o for quick tasks, o3 for hard reasoning sub-problems.
GPT-4o fast tasks o3 hard problems multi-model routing
